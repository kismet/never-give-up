## Parte 1: Uso di Git e GitHub
Le operazione che segueno devono essere eseguite seguendo l'ordine in cui sono presentate.

### Task 1.1: Creazione del proprio branch
Creazione del branch usando `studenti/cognome` dove `cognome` è il cognome dello studente in minuscolo

### Task 1.2: Push del proprio branch sul repository
Assicurarsi di essere collaboratori del progetto inviando all'insegnante (via email, o altro mezzo concordato) il proprio username di GitHub. Una volta ottenuto lo status di collaboratore, procedere con il push del proprio branch sul repository.

### Task 1.3: Implementare il codice
Implementare in locale il codice come indicato dal file CODING.exam.md ed un numero di commit adeguato come da "_best practices_" di git

### Task 1.4: Push finale 
Effettuare il push finale del proprio branch sul repository GitHub

### Task 1.5: Merge dei risultati
Effettuare un branch del branch principale (quello da cui abbiamo creato il proprio branch inizialmente) e rinominarlo in `merging/cognome` (dove `cognome` è il cognome dello studente in minuscolo).

Effettuare il merge del proprio branch in `merging/cognome` e fare la push del branch `merging/cognome` sul repository

### Task 1.6 Risoluzione dei conflitti (BONUS)
Effettuare il merge di  `merging/cognome` su branch principale del repository, quello da cui abbiamo creato il proprio
